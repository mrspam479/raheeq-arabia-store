# 12 — API Contract

All endpoints exposed by the FastAPI backend at `https://api.raheeqarabia.com`. Frontend reaches them either directly (RSC server fetches) or via `app/api/*` proxy routes (browser-initiated POSTs).

## 0. Common headers

| Header | When | Notes |
|---|---|---|
| `Content-Type: application/json` | Always | |
| `Idempotency-Key: <uuid>` | `POST /api/orders` | Required. |
| `X-API-Key: <token>` | `/api/admin/*`, `POST /api/track` | From `BACKEND_API_KEY` env. |
| `X-Forwarded-For` / `X-Real-IP` | from EasyPanel proxy | Used to resolve client IP. |

Errors are returned as:

```json
{ "error": { "code": "VALIDATION_ERROR", "message": "...", "fields": { "phone": "Invalid KSA phone" } } }
```

HTTP status codes:

- `200` OK (GETs and idempotent replays)
- `201` Created (new order)
- `204` No Content (admin status update)
- `400` Bad request (malformed JSON)
- `401` Unauthorized (missing/invalid API key)
- `403` Forbidden (signed upsell token failed)
- `404` Not found (slug/order)
- `409` Conflict (idempotency mismatch)
- `422` Unprocessable Entity (Zod/Pydantic validation)
- `429` Too Many Requests (rate-limited)

## 1. `GET /health` and `GET /ready`

## 4. `POST /api/orders`

Create a COD order.

### Request

Headers: `Idempotency-Key: <uuid>` (required), `Content-Type: application/json`.

Body:

```json
{
  "customer": {
    "full_name": "نورة العتيبي",
    "phone": "+966501234567"
  },
  "lines": [
    {
      "product_slug": "habba-jathr",
      "offer_code": "T3"
    },
    {
      "product_slug": "habba-noura",
      "offer_code": "T1"
    }
  ],
  "tracking": {
    "event_id": "…uuid v4 generated by client…",
    "fbp": "fb.1.1717000000000.123456",
    "fbc": "fb.1.1717000000000.AbCdEfg",
    "ttp": "…",
    "ttclid": "…",
    "sc_click_id": "…",
    "referrer": "https://www.tiktok.com/",
    "landing_url": "https://raheeqarabia.com/p/habba-jathr?utm_source=tiktok",
    "utm": { "source": "tiktok", "medium": "paid", "campaign": "jathr_ksa_v3", "term": null, "content": "ugc_03" },
    "client_user_agent": "Mozilla/5.0 …",
    "client_ip": null
  },
  "honeypot": ""
}
```

The backend ignores any client-supplied `client_ip` and uses the proxy-derived IP.

### Validation rules

- `full_name`: 2–80 chars, must include at least one Arabic letter or two Latin words.
- `phone`: validated via Python `phonenumbers.parse(.., 'SA')` + `is_valid_number` + region must be `SA`. Stored as E.164.
- `lines`: 1–6 items; each `product_slug` must exist and be active; each `offer_code` must exist for that product.
- `tracking.event_id`: required UUID v4 — frontend MUST use the same UUID it used for the `Purchase` pixel call to enable dedup.
- `honeypot`: if non-empty → silently treat as 201 (return fake order, don't persist).

### Response 201

```json
{
  "order": {
    "id": "…uuid…",
    "status": "pending",
    "subtotal_sar": 548,
    "upsell_added_sar": 0,
    "total_sar": 548,
    "currency": "SAR",
    "lines": [
      {
        "product_slug": "habba-jathr",
        "product_name_ar": "حبّة جذر",
        "offer_code": "T3",
        "offer_label_ar": "Glow Kit (الأنصح)",
        "quantity": 3,
        "unit_price_sar": 349,
        "is_upsell": false
      },
      {
        "product_slug": "habba-noura",
        "product_name_ar": "حبّة نورة",
        "offer_code": "T1",
        "offer_label_ar": "علبة",
        "quantity": 1,
        "unit_price_sar": 199,
        "is_upsell": false
      }
    ],
    "created_at": "2026-05-21T17:45:00Z"
  },
  "upsell": {
    "token": "…signed itsdangerous token, TTL 10 min…",
    "sku": "habba-noura",
    "price_sar": 99
  }
}
```

(If the cart already contains every SKU, `upsell` is `null`.)

The signed `upsell.token` is bound to `order.id` and a 10-minute expiry.

### Response 200 (idempotent replay)

Same shape as 201; status is unchanged; if upsell token has expired, a fresh one is reissued.

## 5. `PATCH /api/orders/{id}/upsell`

Attach the 99 SAR upsell.

### Request

```json
{
  "token": "…the signed token from the order response…",
  "sku": "habba-noura"
}
```

Backend verifies `token` (itsdangerous) → checks order id matches the path → checks the SKU is allowed for this order (per the mapping in `05-products.md § 6`) → idempotent: re-applying returns 200 with the already-attached upsell.

### Response 200

```json
{
  "order": {
    "id": "…",
    "subtotal_sar": 548,
    "upsell_added_sar": 99,
    "total_sar": 647,
    "lines": [
      /* existing lines... */
      {
        "product_slug": "habba-noura",
        "product_name_ar": "حبّة نورة",
        "offer_code": "T1",
        "offer_label_ar": "علبة",
        "quantity": 1,
        "unit_price_sar": 99,
        "is_upsell": true
      }
    ]
  }
}
```

## 6. `POST /api/track`

Server-side fan-out endpoint. Frontend calls this **in addition** to firing the web pixels, with the same `event_id`.

### Request

```json
{
  "event_name": "Purchase",
  "event_id": "…uuid v4…",
  "event_time": 1763748300,
  "event_source_url": "https://raheeqarabia.com/p/habba-jathr",
  "user": {
    "phone": "+966501234567",
    "first_name": "نورة"
  },
  "client": {
    "user_agent": "Mozilla/5.0 …",
    "ip": null,
    "fbp": "fb.1.1717000000000.123456",
    "fbc": "fb.1.1717000000000.AbCdEfg",
    "ttp": "…",
    "ttclid": "…",
    "sc_click_id": "…"
  },
  "data": {
    "currency": "SAR",
    "value": 647,
    "contents": [
      { "id": "habba-jathr", "quantity": 3, "item_price": 349 },
      { "id": "habba-noura", "quantity": 2, "item_price": 99 }
    ],
    "num_items": 5,
    "content_type": "product",
    "order_id": "…order uuid…"
  }
}
```

`event_name` ∈ `ViewContent | AddToCart | InitiateCheckout | Purchase | UpsellAccept`.

### Response 202

```json
{ "queued": true }
```

Backend enqueues 3 platform-specific deliveries. Failures retry in background.

`POST /api/track` requires header `X-API-Key: <BACKEND_API_KEY>`. The Next.js proxy route (`app/api/track/route.ts`) injects this server-side so the key never reaches the browser.

## 7. `GET /api/admin/orders`

Auth: `X-API-Key`.

Query params: `status`, `q` (search name/phone), `from`, `to`, `limit`, `cursor`.

Response: paginated `{ items: [Order], next_cursor }`.

## 8. `PATCH /api/admin/orders/{id}`

Auth: `X-API-Key`. Body: `{"status": "confirmed" | "shipped" | "delivered" | "returned" | "cancelled", "address_line": "...", "city": "...", "notes": "..."}`. Any subset.

Returns `204`.

## 9. Rate limiting

- `POST /api/orders` → 6 / min / IP.
- `POST /api/track` → 60 / min / IP (it is internal but still rate-limited).
- Admin endpoints: 60 / min / IP.

Exceeded → `429 { "error": { "code": "RATE_LIMITED", "message": "أعيدي المحاولة بعد قليل." } }`.

## 10. CORS

Allowed origins: `https://raheeqarabia.com` and `http://localhost:3000` (dev). Allowed methods: `GET, POST, PATCH, OPTIONS`. Allowed headers: `Content-Type, Idempotency-Key, X-API-Key`. `Access-Control-Max-Age: 86400`.

## 11. OpenAPI

FastAPI emits `/openapi.json` and `/docs` automatically. In production we **disable** `/docs` and `/redoc` (set `docs_url=None, redoc_url=None` when `APP_ENV=production`). `/openapi.json` is also disabled in prod.

## 12. Versioning

URLs are unversioned in v1. If a breaking change is needed later, introduce `/v2/...` and keep v1 alive 60 days.
